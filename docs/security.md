# 安全说明

## 是否包含个人敏感信息

当前仓库不包含个人敏感信息。

发布前已检查并避免纳入：

- 微信公众号后台 token。
- Cookie。
- 账号 ID。
- 已导出的 CSV/JSON 数据。
- 文章内容样本。
- 本机绝对路径。
- 浏览器历史记录或配置。

`.gitignore` 已排除常见导出文件：

```text
wechat-mp-recent-*.csv
wechat-mp-recent-*.json
outputs/
downloads/
```

## 权限模型

脚本和扩展只在以下域名下运行：

```text
https://mp.weixin.qq.com/*
```

Chrome 扩展声明的 `host_permissions` 也只覆盖该域名。

## Cookie 与 token

脚本不会主动读取：

- `document.cookie`
- 浏览器 Cookie 存储
- 浏览器密码
- 浏览器历史记录

API 模式会在当前已登录页面内发起同源 `fetch` 请求。浏览器会自动携带登录态，这是网页正常请求机制；脚本不会把这些凭据保存、展示或导出。

URL 中的后台 token 只用于构造同源列表请求，不写入导出文件。

## 数据存储

采集到的数据会暂存在当前浏览器的 `localStorage` 中，key 为项目私有前缀。点击面板中的 `Clear` 会清空这部分缓存。

导出文件由浏览器下载到本机，不会上传到外部服务。

## 网络行为

脚本只请求：

- 微信公众号后台同源列表接口。
- 已采集到的微信公众号文章公开链接。

脚本不请求第三方统计、日志或云服务。

## 风控与平台规则

本项目用于导出自己有权限访问的数据，不包含绕过登录、验证码、权限校验、指纹检测或平台风控的功能。

脚本默认使用低频、单线程、失败退避的方式降低请求压力，但这不是规避检测机制的设计。使用者应遵守微信公众平台相关规则。

## 发布前自查命令

可以用以下命令做本地敏感信息扫描：

```bash
rg -n "token=|cookie|Cookie|bizuin|slave_|openid|pass_ticket|/Users/" .
```

如果出现真实凭据、导出数据或本机路径，请在提交前移除。
